# AGENTS.md

Guidance for AI coding agents working on `sosialhjelp-soknad`.

## Project

Citizen-facing Next.js frontend for applying for social welfare (sosialhjelp) in Norway.
Built by NAV team Digisos. Base path: `/sosialhjelp/soknad`. Supports locales: `nb`, `nn`, `en`.

## Commands

```bash
pnpm install          # Install dependencies
pnpm dev              # Dev server on port 3001 (requires local backend, see README)
pnpm build            # Production build
pnpm test             # Vitest unit tests (watch mode)
pnpm test --run       # Vitest unit tests (CI / single run)
pnpm e2e              # Playwright E2E tests (headless)
pnpm e2e:ui           # Playwright E2E tests (UI mode)
pnpm lint             # ESLint
pnpm tsc              # TypeScript type-check (no emit)
pnpm run fetch-api    # Fetch OpenAPI specs from local backend → soknad-api-*.json
pnpm run orval        # Regenerate API client code from OpenAPI specs
```

Use `pnpm` exclusively. `npm` and `yarn` are blocked by a `preinstall` hook.

## Directory layout

```
src/
  app/              # Next.js App Router root
    [locale]/       # Locale-prefixed routes (nb/nn/en)
    api/            # BFF proxy API routes (never call soknad-api directly from browser)
  sider/            # Form step implementations
    01-personalia/  # Step 1 – personal info
    02-begrunnelse/ # Step 2 – reason for application
    ...             # Steps 03–09 follow the same pattern
    09-oppsummering/
    kort/           # Shortened form variant
  lib/
    api/            # Axios instance, SSR fetch, error handling
    components/     # Shared UI (SkjemaSteg step shell, file upload, etc.)
    hooks/          # Custom hooks: data/, dokumentasjon/, auth/
    providers/      # React context providers (DigisosContext, analytics, etc.)
    config.ts       # Per-environment config and feature flags
    validering.ts   # Validation helpers
  generated/        # ⚠️ Auto-generated by Orval — DO NOT EDIT
  locales/          # Typed translation strings (nb/nn/en)
  i18n/             # next-intl routing config
e2e/                # Playwright end-to-end tests
.nais/              # Kubernetes deployment manifests (NAIS/GCP)
```

## Architecture

- **BFF proxy**: `src/app/api/` proxies all browser requests to `sosialhjelp-soknad-api`. Never call the backend API directly from client code.
- **Orval-generated clients**: All API calls use generated TanStack Query hooks (CSR) or `authenticatedFetch` (SSR). Four targets: `old` and `new` API × CSR and SSR.
- **Dual API migration**: Old and new API clients coexist under `src/generated/`. Pages migrate incrementally from old to new.
- **Form steps**: Each step lives in a numbered `sider/0N-name/` directory and uses the `SkjemaSteg` component system for layout, navigation, and error handling.
- **State**: TanStack React Query for server state. `DigisosContext` for session/soknad identity.
- **Auth**: Wonderwall sidecar handles IDporten OAuth2. Tokens stay server-side.

## Code conventions

- **TypeScript strict**: `noUnusedLocals`, `noUnusedParameters` are on. Fix all TS errors before committing.
- **Translation keys are typed**: Use keys from `src/locales/types.ts`. Do not use arbitrary strings with `t()`.
- **React Query for data fetching**: Use generated hooks or existing custom hooks in `src/lib/hooks/data/`. Do not use raw `fetch`/`axios` in components.
- **Zod + React Hook Form**: Use Zod schemas with `@hookform/resolvers` for form validation.
- **Pino logging**: Import logger via `@log/` path alias (`import logger from "@log/index"`).
- **Feature flags**: Read from `useConfigFeatureFlags()` — do not read `config.ts` directly in components.

## Testing

- **Unit tests (Vitest)**: Co-located with source files (`*.test.ts`). Run with `pnpm test --run`.
- **E2E tests (Playwright)**: Located in `e2e/`. Require either a running dev server or `pnpm build:e2e` + `pnpm start`. Run with `pnpm e2e`.
- Write unit tests for utilities, hooks, and validation logic. Write E2E tests for full form step flows.

## Critical constraints

1. **Never edit files in `src/generated/`**. They are regenerated by `pnpm run orval`. Make changes to the OpenAPI spec or `orval.config.ts` instead, then regenerate.
2. **Run `pnpm run orval` after changing `soknad-api-old.json` or `soknad-api-new.json`**.
3. **Use `pnpm` only** — `npm`/`yarn` will fail at `preinstall`.
4. **All linting must pass**: Husky enforces max 1 ESLint warning and Prettier formatting on commit.
5. **Locale messages must be added to all three languages** (`nb`, `nn`, `en`) under `messages/` and typed in `src/locales/`.
